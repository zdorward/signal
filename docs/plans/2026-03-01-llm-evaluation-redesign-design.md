# LLM Evaluation Redesign

## Overview

Redesign the evaluation system to use employer-defined custom questions with AI-generated, editable criteria. Replace the current fixed 5-category rubric with a flexible question-based approach.

## Problems with Current System

1. **Fixed 5 categories not editable** — Rubric generated at challenge creation and locked
2. **URL evaluation unreliable** — System only checks if URL resolves, cannot analyze actual content (e.g., cannot detect that a portfolio has /books and /projects pages)
3. **Video not analyzed** — Video path passed to Claude as text, but content not evaluated
4. **AI scoring too strict** — Tendency to never give 5/5 scores

## Design

### Challenge Creation Flow

**Input from employer:**
- Role description (existing field)
- General challenge requirements (new field, optional) — e.g., "Should include a demo link and video walkthrough. Focus on AI product thinking."

**AI generates:**
- 3-5 custom questions (e.g., "Why Wealthsimple?", "What decision must remain human?")
- 1-3 evaluation criteria per question
- Standard fields: "Demo URL" (always included, pass/fail) and "Video upload" (optional)

**Editing interface:**
- Single screen showing all questions with their criteria nested below
- Employer can:
  - Edit question text
  - Add/remove questions
  - Edit criterion text
  - Add/remove criteria per question
  - Reorder questions
- "Regenerate" button to get fresh AI suggestions (replaces everything)

### Candidate Submission Experience

**What the candidate sees:**

1. **Challenge intro** — Brief context about the role/challenge (generated by AI, editable by employer)

2. **Custom questions** — Each question displayed as a text field
   - Character/word limit per question (configurable by employer, default 500 words)
   - Questions displayed in order set by employer

3. **Demo URL field** — Required, with helper text: "Link to your working demo, portfolio, or project"

4. **Video upload** — Optional (unless employer marks required)
   - Same specs as today: MP4/WebM/MOV, 512MB max

5. **Submit button** — Triggers evaluation

### Evaluation Logic

**Scoring structure:**
- Each criterion scored 1-5
- URL check: pass/fail (resolves + AI confirms relevance to challenge)
- Video: not scored by AI in initial phase

**Scoring scale:**

| Score | Meaning |
|-------|---------|
| 5 | Exceeds expectations — thoughtful, specific, demonstrates real understanding |
| 4 | Meets expectations — solid answer that addresses the question well |
| 3 | Adequate — answers the question but lacks depth or specificity |
| 2 | Weak — partially addresses the question, missing key elements |
| 1 | Poor — doesn't answer the question, generic, or off-topic |

**AI prompt calibration (to fix "never gives 5/5" problem):**
1. Explicitly define what each score means
2. Include examples of 5/5 answers — "A 5 doesn't require perfection, it requires thoughtfulness and specificity"
3. Instruct AI to evaluate against the criterion only, not impose unstated requirements
4. Include calibration check: "If this answer directly addresses the criterion with specific, relevant detail, score it 4 or 5"

**URL validation:**
1. Check URL returns HTTP 200
2. Fetch page content
3. AI does quick relevance check — "does this look like it matches what the challenge asked for?"
4. Return pass/fail with brief note

**Evaluation output per submission:**
- Score (1-5) + reasoning for each criterion
- URL status: pass/fail + brief note
- Overall summary bullets (3-5 takeaways)
- `worth_human_attention` flag
- Rejection draft

**Aggregation:**
- Average score across all criteria (displayed to employer)
- Individual criterion scores visible in detail view

### Employer Dashboard

**Submissions list view:**
- Candidate name
- Average score (1-5, one decimal place, e.g., "4.2")
- URL status: checkmark or X
- `worth_human_attention` badge
- Submitted date

**Submission detail view:**
- Candidate's answers to each question
- Per-criterion scores with AI reasoning
- URL status + AI note (e.g., "Portfolio site with projects page, matches challenge requirements")
- Video player (if uploaded) — employer watches manually for now
- Summary bullets
- Rejection draft (copyable)

**Challenge management:**
- Edit questions/criteria anytime (affects future evaluations only)
- View challenge as candidate would see it (preview mode)

### Data Model Changes

**challenges table:**
- Remove: `challenge_text`, `rubric_json`
- Add: `intro_text` (string), `questions_json` (array of question objects)

**Question object structure:**
```json
{
  "id": "uuid",
  "text": "Why Wealthsimple?",
  "order": 1,
  "word_limit": 500,
  "criteria": [
    {
      "id": "uuid",
      "text": "Demonstrates specific knowledge of Wealthsimple's mission, products, or culture",
      "order": 1
    },
    {
      "id": "uuid",
      "text": "Shows genuine connection between candidate's values and company",
      "order": 2
    }
  ]
}
```

**submissions table:**
- Remove: `written_explanation`
- Add: `answers_json` (array of answer objects)

**Answer object structure:**
```json
{
  "question_id": "uuid",
  "text": "I want to work at Wealthsimple because..."
}
```

**evaluations table:**
- Replace `rubric_scores_json` with `criterion_scores_json`
- Add: `url_passed` (boolean), `url_notes` (string)

**Criterion score object structure:**
```json
{
  "criterion_id": "uuid",
  "question_id": "uuid",
  "score": 4,
  "reasoning": "Candidate demonstrates clear understanding..."
}
```

### Migration

- Replace old rubric system entirely
- No backwards compatibility needed
- Existing challenges can be archived or deleted

## Future: Video Analysis

**Not in scope for initial implementation.** Document requirements for future phase:

### Requirements

- Video analyzed by vision model (Claude with vision or similar)
- Employer can optionally tie video to a specific question (e.g., "Walk us through your demo")
- If tied to a question, video analysis contributes to that question's criteria scores
- If standalone, video gets its own criteria (AI-generated, employer-editable)

### Technical Considerations

- Video length limits (recommend 5 min max for cost/processing)
- Processing time may be longer than text evaluation
- Cost per evaluation increases significantly with video
- May need async processing with status updates
- Consider extracting key frames vs. full video analysis

### Open Questions

- Should video be required or optional per challenge?
- How to handle very long videos?
- Should we support screen recordings specifically?
- Integration with existing UploadThing infrastructure
